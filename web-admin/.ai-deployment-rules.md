# CRITICAL DEPLOYMENT WORKFLOW (VERCEL + GITHUB)

**Hobby Tier Vercel Blokir GitHub Push!**
Karena repository `web-admin` terhubung dengan akun korporat/bukan hobby di GitHub yang berbenturan dengan model pricing Vercel, push ke GitHub secara **otomatis tidak akan ter-deploy di Vercel**. Vercel akan memblokirnya dengan pesan "Deployment Blocked".

### PROSEDUR WAJIB SETIAP KALI DEPLOY WEB KE LIVE:
Setiap agen AI (Antigravity, Claude Code, dsb) harus menjalankan dua langkah perintah berikut secara berurutan saat User meminta "Push/Deploy":

**Langkah 1: Backup ke GitHub**
```bash
git add .
git commit -m "Deskripsi perubahan"
git push
```

**Langkah 2: Deployment Paksa ke Vercel (CLI)**
Membypass bloking dengan melakukan publish paksa dari lokal:
```bash
npx vercel --prod --yes
```

**Penting:** JANGAN PERNAH hanya melakukan `git push` lalu berharap website Vercel akan terupdate otomatis. Anda WAJIB memanggil `npx vercel --prod --yes` di akhir setiap pekerjaan deploy.
